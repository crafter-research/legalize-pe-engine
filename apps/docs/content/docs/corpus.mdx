El corpus vive en [`crafter-research/legalize-pe`](https://github.com/crafter-research/legalize-pe):
una norma por archivo Markdown con frontmatter YAML, organizadas por jurisdicción
(`pe/` nacional, `pe-xxx/` regional). Cada reforma es un commit, así que el
historial de una norma es su log de git.

## Cobertura

- **21,244 normas**, ~98% con texto completo.
- **27 jurisdicciones**: nacional (`pe`) + 26 regiones (`pe-are`, `pe-cus`, …).
- Fuente: SPIJ (nacional) + portales regionales de gob.pe. Sin texto inventado.

## Esquema (frontmatter)

```yaml
identifier: RM-0702-2026-IN
title: Aprueban el Texto Integrado Actualizado del ROF del Ministerio del Interior
rank: resolucion_ministerial        # tipo de norma
publication_date: '2026-05-26'
jurisdiction: pe                    # pe (nacional) o pe-xxx
scope: Nacional
status: in_force
source: 'https://spij.minjus.gob.pe/...'   # fuente oficial
```

El cuerpo (Markdown) sigue al frontmatter.

## Identificadores

Los `identifier` pueden colisionar entre jurisdicciones, así que el id canónico
es el **path relativo sin `.md`** (ej. `pe/RM-0702-2026-IN`). Ese id es el que
usan la API, el MCP y el CLI.
